### Session — 07-20 (v0.99.1, v0.99.2)

**Focus**: Package ideation reliability fixes and SSO auth latency hardening for PR.

- **Root Cause**: Ideation advance responses did not expose the saved user-message marker and completed sessions could return conflicts on repeat advance calls. SSO token introspection also made a remote HTTP call on every protected request, so slow SSO responses dominated endpoint latency.
- **Changes**:
  - Hardened ideation structured-output normalization, including question/recommendation padding, summary draft coercion, and read/write repair for raw JSON agent messages.
  - Added `since_message_id` incremental message fetching and `user_message_id` on advance responses.
  - Made completed ideation advance calls idempotent by returning the existing PRD run details.
  - Added short-TTL SSO introspection caching, cache clearing for tests, token-expiry clamping, bounded cache size, and configurable introspection timeout.
  - Updated environment-variable and ideation API documentation.
